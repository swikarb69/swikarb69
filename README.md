<summary><b>в—Ү &nbsp; workflow setup вҖ” snake + pacman</b></summary>

**Before anything else вҖ” enable private contributions:**
Go to **GitHub вҶ’ Settings вҶ’ Public profile** and turn on **"Include private contributions on my profile."** Without this, the snake, pacman, and streak-stats widgets below can only see your *public* squares вҖ” which is why they were undercounting (103 vs. your real 311) and why pacman only looked partially filled.

Add this to `.github/workflows/snake.yml` in your profile repo:

```yaml
name: Generate Snake + Pac-Man

on:
  schedule:
    - cron: "0 */12 * * *"
  workflow_dispatch:
  push:
    branches: [main]

jobs:
  generate:
    permissions:
      contents: write
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4

      - name: Generate Snake graph
        uses: Platane/snk/svg-only@v3
        with:
          github_user_name: swikarb69
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark&color_snake=%23A78BFA

      - name: Generate Pac-Man graph
        uses: abozanona/pacman-contribution-graph@main
        with:
          github_user_name: swikarb69
          games: 'pacman'

      - name: Push to output branch
        uses: crazy-max/ghaction-github-pages@v3
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

After merging, trigger it manually once from the **Actions** tab (`workflow_dispatch`), then check the `output` branch to confirm these four files exist before trusting the README links:
`github-contribution-grid-snake.svg`, `github-contribution-grid-snake-dark.svg`, `pacman-contribution-graph.svg`, `pacman-contribution-graph-dark.svg`.
